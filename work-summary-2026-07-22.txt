一、安全漏洞修复（P0）

  - TOCTOU 竞态条件修复
    validateFilePath 改为返回已解析的真实路径(realpath)，所有内部 readFile
    调用使用验证后的路径，消除路径校验与文件读取之间的符号链接交换窗口。

  - export_index 路径约束
    自定义 outputPath 参数限定在 dbPath 目录内，阻止任意文件写入。

  - ingest_data 大小限制
    新增 MAX_INGEST_DATA_SIZE = 100MB 上限，防止通过超大内联内容耗尽内存。

  - 代理 URL 日志脱敏
    控制台日志中的代理 URL 仅记录 protocol://host，密码/用户名不再暴露。

  - server.json 版本同步
    修复长期存在的 server.json (0.18.9) 与 package.json (0.19.2) 版本不一致问题。

三、CI/CD 自动化建设

    tag push 自动发布到 npm registry，替换之前需要人工 OTP 的手动流程。

  - 修复 publish-mcp-registry.yml
    workflow_dispatch 手动触发时版本号从 package.json 读取（修复
    refs/heads/main 写入 server.json 的 bug）；新增 npm 版本就绪轮询

  - 覆盖率 job

  - 稳定性修复

      解决 vitest isolate=false 下的模块冲突

四、新功能：health_check MCP 工具

  问题：用户 npx 首次运行遇到模型下载失败、LanceDB 不可访问、
  BASE_DIR 不存在等问题时，只能翻 README FAQ 排查，体验很差。

  方案：新增 health_check 工具，主动探测 4 个关键子系统：
    • Config — 逐个检查 BASE_DIR 读写权限
    • Embedder — 实际执行一次轻量嵌入验证模型可用
    • LanceDB — 读取文件列表确认数据库可访问
    • Cache — 检查模型缓存目录可写性

  输出结构化 pass/fail/warn 结果 + 自然语言摘要 + 每个失败项的具体
  修复建议。与 status 工具的区别：status 只 dump 静态元数据，
  health_check 主动探测并翻译成人话。
